# EXPERIMENT PACKAGE — Echorouk Editorial OS

هذا الملف هو نقطة الدخول لوضع المشروع في **مرحلة التجربة** مع مرجع سريع للتشغيل والمتابعة وحل الأعطال.

## 1) الملفات المرجعية الأساسية
- `docs/INSTRUCTURE_PLATFORM.md`:
  المرجع التقني/التحريري الرئيسي.
- `docs/PLATFORM_CONTENT_MAP.md`:
  خريطة المحتوى الكامل للمنصة (Backend/Frontend/Workflow/RBAC).
- `docs/EXPERIMENT_SMOKE_TEST_PLAN.md`:
  سيناريوهات التجربة النهائية (Smoke + Role-based).
- `docs/TROUBLESHOOTING_PLAYBOOK.md`:
  دليل الأعطال الأكثر تكرارًا مع أوامر الإصلاح المباشر.
- `docs/OPERATIONS_QUICK_COMMANDS.md`:
  أوامر GitHub + السيرفر + التحقق التشغيلي السريع.

## 2) تعريف جاهزية التجربة
المنصة تعتبر جاهزة للتجربة عندما:
- تسجيل الدخول يعمل لكل دور.
- صفحة الأخبار + المحرر الذكي + التنبيهات تعمل بدون 500.
- دورة الخبر تصل إلى `ready_for_manual_publish` بدون نشر تلقائي.
- رئيس التحرير يرى طابور الاعتماد.
- السوشيال يرى الأخبار المعتمدة فقط ونسخ الـ Copy/Paste.

## 3) ترتيب العمل الموصى به يوميًا
1. فحص الصحة: `/health`.
2. فحص التنبيهات: `/api/v1/dashboard/notifications`.
3. فحص الأخبار حسب الحالات الأساسية.
4. تشغيل `EXPERIMENT_SMOKE_TEST_PLAN.md` على عينة أخبار.
5. عند خطأ: الرجوع مباشرة إلى `TROUBLESHOOTING_PLAYBOOK.md`.

## 4) قاعدة تشغيل مهمة
- النشر النهائي يبقى يدويًا 100%.
- أي خبر لا يمر بوابات الجودة/التحقق/السياسة التحريرية لا ينتقل إلى الجاهزية النهائية.
